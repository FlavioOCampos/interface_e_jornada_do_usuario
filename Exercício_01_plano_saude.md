# 🧪 Avaliação por Pares – Teste de Usabilidade (Exercício 01)

Nesta atividade, os projetos de **Wireframe Mobile** serão avaliados pelos colegas da turma através de **testes de usabilidade**.

O objetivo é verificar se a interface criada permite que a tarefa principal seja realizada com **clareza, facilidade e autonomia**, além de identificar **pontos de melhoria** na experiência do usuário.

---

# 🔍 2. Escolhendo Projetos para Testar

Após todos publicarem seus links:

- Cada estudante deve **testar 3 projetos de colegas**.
- Antes de iniciar o teste, comentar **"TESTE"** na postagem do colega.

⚠ **Importante**

Cada projeto pode receber **no máximo 3 testes**.  
Antes de escolher um projeto, verifique se já existem três comentários **"TESTE"**.

Se houver, escolha outro projeto.

---

# 🧾 3. Realizando o Teste

Após testar a interface do colega, registre sua avaliação no **Formulário de Avaliação** disponibilizado pelo professor.

Cada estudante deverá preencher o formulário **3 vezes**, uma para cada projeto avaliado.

No formulário deverão ser informados:

- Qual projeto foi testado
- Avaliação baseada nas **rubricas**
- Observações sobre a **experiência de uso**

---

# ✅ 4. Requisitos para Obter Nota

Para que a atividade seja considerada completa, o estudante deverá:

- ✔ Testar **3 interfaces de colegas**
- ✔ Preencher o formulário **3 vezes**
- ✔ Receber **pelo menos 3 avaliações** no próprio projeto

❗ Estudantes que não realizarem os testes ou não receberem o número mínimo de avaliações **não terão pontuação registrada**.

---

# 💡 5. Dica Importante

Durante o teste, tente se colocar no lugar de um **usuário real**.

- Siga o fluxo proposto pela interface  
- Observe se as ações são claras  
- Registre dificuldades ou dúvidas encontradas  

O objetivo da atividade não é apenas atribuir notas, mas **identificar melhorias e aprender com as soluções desenvolvidas pelos colegas**.
